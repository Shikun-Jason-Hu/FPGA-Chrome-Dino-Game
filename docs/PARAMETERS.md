# Parameter Reference

This document lists the main constants and configuration values used in the FPGA Chrome Dino Game.

Most values are currently written as `localparam` constants inside each SystemVerilog module. To change the game behavior, edit the value in the corresponding module and recompile the Quartus project.

> Note: For a more reusable HDL design, frequently changed `localparam` values can later be converted into module `parameter`s.

---

## Top-Level Configuration

**File:** `rtl/DE1_SoC.sv`

| Setting | Current Value | Description |
|--------|---------------|-------------|
| System clock | `CLOCK_50` | 50 MHz onboard DE1-SoC clock |
| Game tick | `divided_clocks[21]` | Slower frame/game update clock generated by `clock_divider` |
| Reset input | NES `start` button | Used to reset the game |
| Jump input | NES `up` button | Makes the Dino jump |
| Duck input | NES `down` button | Makes the Dino duck |
| VGA output | `VGA_R/G/B`, `VGA_HS`, `VGA_VS` | Video output signals |

### Suggested changes

- Change `divided_clocks[21]` to another divided clock index to make the game update faster or slower.
- Change controller button mappings in the `n8_driver` instantiation if different controls are desired.

## Dino Controller

**File:** `rtl/dino_controller.sv`

Controls the Dino finite-state machine, jump physics, ducking behavior, and animation timing.

| Constant | Current Value | Description | Effect of Changing |
|----------|---------------|-------------|--------------------|
| `DINO_X` | `10'd50` | Fixed horizontal position of the Dino | Moves the Dino left or right on the screen |
| `GROUND_Y` | `8'd178` | Dino standing y-position | Changes where the Dino stands vertically |
| `DUCK_Y` | `8'd187` | Dino y-position while ducking | Changes ducking height/position |
| `JUMP_VELOCITY` | `-15` | Initial upward jump velocity | Larger negative value makes the Dino jump higher |
| `GRAVITY` | `9'd1` | Downward acceleration added each tick | Larger value makes the Dino fall faster |
| `ANIMATION_FRAME` | `4'd8` | Number of ticks before animation frame changes | Smaller value makes running animation faster |
| `STATUS_RUN` | `2'd0` | Dino running status code | Must match renderer/hitbox expectations |
| `STATUS_JUMP` | `2'd1` | Dino jumping status code | Must match renderer/hitbox expectations |
| `STATUS_DUCK` | `2'd2` | Dino ducking status code | Must match renderer/hitbox expectations |

### Suggested changes

- Increase `JUMP_VELOCITY` magnitude for a higher jump.
- Increase `GRAVITY` for a faster fall and harder gameplay.
- Change `ANIMATION_FRAME` to tune Dino animation speed.
- Be careful changing status values because other modules depend on these encodings.

## Obstacle Controller

**File:** `rtl/obstacle_controller.sv`

Handles obstacle positions, obstacle movement, despawning, and ground scrolling.

| Constant | Current Value | Description | Effect of Changing |
|----------|---------------|-------------|--------------------|
| `SCREEN_W` | `9'd320` | Logical game width | Should match renderer `GAME_WIDTH` |
| `SCROLL_SPEED` | `9'd3` | Pixels moved left per game tick | Higher value makes obstacles and ground move faster |
| `GROUND_TEX_W` | `9'd200` | Ground texture wrap width | Controls when the scrolling ground loops |
| `TYPE_NONE` | `2'd0` | No obstacle type code | Must match spawner/renderer/hitbox |
| `TYPE_CACTUS` | `2'd1` | Cactus obstacle type code | Must match spawner/renderer/hitbox |
| `TYPE_BIRD` | `2'd2` | Bird/pterosaur obstacle type code | Must match spawner/renderer/hitbox |
| `CACTUS_Y` | `8'd164` | Cactus vertical spawn position | Changes cactus height/placement on ground |
| `BIRD_LOW` | `8'd172` | Low bird y-position | Changes low bird flight height |
| `BIRD_HIGH` | `8'd165` | High bird y-position | Changes high bird flight height |
| `MAX_OBS` | `2` | Maximum obstacles on screen | More obstacles require wider output/interface changes |

### Suggested changes

- Increase `SCROLL_SPEED` for harder gameplay.
- Change `BIRD_HIGH` and `BIRD_LOW` to tune whether the Dino must jump or duck.
- Do not change obstacle type encodings unless all related modules are updated.

## Obstacle Spawner

**File:** `rtl/obstacle_spawner.sv`

Controls when new obstacles are spawned.

| Constant | Current Value | Description | Effect of Changing |
|----------|---------------|-------------|--------------------|
| `MIN_GAP` | `7'd90` | Minimum delay between obstacle spawns | Smaller value spawns obstacles more often; larger value makes the game easier |

The final spawn delay is:

```text
MIN_GAP + spawn_gap
```

where `spawn_gap` comes from the LFSR module.

### Suggested changes

- Lower `MIN_GAP` to make obstacles appear more frequently.
- Raise `MIN_GAP` to make the game easier.
- If modifying the LFSR output width, also update the `spawn_gap` input width.

## Hit Box Detector

**File:** `rtl/hit_box_detector.sv`

Defines collision box sizes for the Dino and each obstacle type.

| Constant | Current Value | Description | Effect of Changing |
|----------|---------------|-------------|--------------------|
| `DINO_RUN_WIDTH` | `30` | Dino collision width while running/jumping | Larger value makes collisions easier |
| `DINO_RUN_HEIGHT` | `16` | Dino collision height while running/jumping | Larger value makes collisions easier |
| `DINO_DUCK_WIDTH` | `30` | Dino collision width while ducking | Larger value makes ducking less forgiving |
| `DINO_DUCK_HEIGHT` | `16` | Dino collision height while ducking | Smaller value makes ducking more forgiving |
| `CACTUS_WIDTH` | `18` | Cactus collision width | Larger value makes cactus collisions easier |
| `CACTUS_HEIGHT` | `36` | Cactus collision height | Larger value makes cactus collisions easier |
| `PTEROSAUR_WIDTH` | `22` | Pterosaur collision width | Larger value makes bird collisions easier |
| `PTEROSAUR_HEIGHT` | `14` | Pterosaur collision height | Larger value makes bird collisions easier |

### Suggested changes

- Use slightly smaller hitboxes than the visible sprites for fairer gameplay.
- Keep hitbox dimensions consistent with sprite dimensions in `renderer.sv`.

## Renderer

**File:** `rtl/renderer.sv`

Controls game dimensions, sprite IDs, ROM dimensions, ground rendering, and game-over rendering.

### Game Dimensions

| Constant | Current Value | Description |
|----------|---------------|-------------|
| `GAME_WIDTH` | `320` | Logical game width |
| `GAME_HEIGHT` | `240` | Logical game height |

### Sprite IDs

These IDs determine which sprite row block is read from `sprite_rom`.

| Constant | Current Value | Sprite |
|----------|---------------|--------|
| `SPR_DINO_RUN1` | `0` | Dino run frame 1 |
| `SPR_DINO_RUN2` | `1` | Dino run frame 2 |
| `SPR_DINO_JUMP` | `2` | Dino jump frame |
| `SPR_DINO_DUCK1` | `3` | Dino duck frame 1 |
| `SPR_DINO_DUCK2` | `4` | Dino duck frame 2 |
| `SPR_DINO_HIT` | `5` | Dino hit/dead frame |
| `SPR_CACTUS` | `6` | Cactus obstacle |
| `SPR_PTERO_UP` | `7` | Pterosaur wing-up frame |
| `SPR_PTERO_DN` | `8` | Pterosaur wing-down frame |

### ROM and Sprite Dimensions

| Constant | Current Value | Description | Effect of Changing |
|----------|---------------|-------------|--------------------|
| `SPRITE_W` | `32` | Width of each sprite ROM slot | Must match generated sprite ROM width |
| `SPRITE_H` | `40` | Height of each sprite ROM slot | Must match generated sprite ROM height |
| `GROUND_Y` | `175` | Ground vertical render position | Moves ground up/down |
| `GROUND_W` | `320` | Ground ROM width | Must match `ground_rom.hex` row width |
| `GROUND_H` | `8` | Ground ROM height | Must match `ground_rom.hex` row count |
| `GAMEOVER_W` | `100` | Game-over image width | Must match `game_over.hex` row width |
| `GAMEOVER_H` | `50` | Game-over image height | Must match `game_over.hex` row count |

### ROM Files

The renderer currently loads:

```systemverilog
$readmemh("sprite_rom.hex", sprite_rom);
$readmemh("ground_rom.hex", ground_rom);
$readmemh("game_over.hex", gameover_rom);
```

If the ROM files are moved into a different directory, update these paths based on the location of the Quartus project file.

Example:

```systemverilog
$readmemh("rom/sprite_rom.hex", sprite_rom);
$readmemh("rom/ground_rom.hex", ground_rom);
$readmemh("rom/game_over.hex", gameover_rom);
```

or, if the Quartus project is inside a `quartus/` folder:

```systemverilog
$readmemh("../rom/sprite_rom.hex", sprite_rom);
$readmemh("../rom/ground_rom.hex", ground_rom);
$readmemh("../rom/game_over.hex", gameover_rom);
```

### Suggested changes

- If adding new sprites, update both the sprite ROM generation script and the sprite ID constants.
- If changing `SPRITE_W` or `SPRITE_H`, regenerate `sprite_rom.hex` using the same dimensions.
- If changing ground or game-over dimensions, regenerate the corresponding ROM files.

## Graphic Driver

**File:** `rtl/graphic_driver.sv`

The graphic driver connects the renderer to the VGA driver.

| Setting | Current Value | Description |
|---------|---------------|-------------|
| VGA logical width | `320` | Passed to `video_driver` as `.WIDTH(320)` |
| VGA logical height | `240` | Passed to `video_driver` as `.HEIGHT(240)` |

### Suggested changes

- Keep `.WIDTH(320)` and `.HEIGHT(240)` consistent with `renderer.sv`.
- If changing display resolution, update both `graphic_driver.sv` and `renderer.sv`.

## Recommended Future Refactor

For easier customization, convert important `localparam` values into module `parameter`s.

Example:

```systemverilog
module dino_controller #(
    parameter DINO_X = 10'd50,
    parameter GROUND_Y = 8'd178,
    parameter signed JUMP_VELOCITY = -15,
    parameter GRAVITY = 9'd1
)(
    // ports...
);
```

Then the top-level module can override values during instantiation:

```systemverilog
dino_controller #(
    .JUMP_VELOCITY(-18),
    .GRAVITY(9'd2)
) dino_ctrl (
    // connections...
);
```

This makes the game easier to tune without editing each module directly.
